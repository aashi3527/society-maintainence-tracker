# Society Maintenance Tracker (Simplified Version)

A lightweight Node/Express and Vanilla HTML/CSS/JS application to track society maintenance complaints and notices. It runs with zero compilation steps or build configurations.

---

## 🚀 How to Run the App

1. **Navigate** into the project folder:
   ```bash
   cd society-maintenance-tracker
   ```
2. **Install Dependencies**:
   ```bash
   npm install
   ```
3. **Start the Server**:
   ```bash
   node server.js
   ```
4. **Access the App**: Open [http://localhost:3000](http://localhost:3000) in your browser.

## 📬 Testing Email Notifications (Mock Mailbox)
A floating **Mock Mailbox button 📬** is visible at the bottom right. 
- The drawer automatically displays a list of emails generated by the system (e.g. status updates or important notices).
- This allows you to evaluate the notification system immediately without setting up actual SMTP details.
- To configure actual SMTP emails, add standard variables in a local `.env` file (see `.env.example`).

---

## 📋 Database Tables (SQLite)

- **`users`**: Registers accounts with columns for `username`, `email`, `role`, and `unit_number`.
- **`complaints`**: Tracks complaints raised by residents including category, description, photo attachment URL, status, and priority.
- **`status_history`**: Holds a log of every status change (timestamp, actor, optional note).
- **`notices`**: Notice board announcements (important notices are pinned).
- **`settings`**: Configures the overdue threshold in days.
- **`mock_emails`**: Logs outgoing notifications for local testing.
